# Project Memory

- Project structure: `backend/` is the FastAPI/Python side (`main.py` API routes, `models.py`, `paths.py`, `pdf_service.py`, `reflow_service.py`, `tts_service.py`, `orpheus_service.py`, `qwen_multilingual_tts_service.py`, `tts_queue.py`; generated/runtime data lives under `backend/data/`, `backend/uploads/`, and `backend/audio_cache/`; ONNX voice/model files live under `backend/models/`).
- Project structure: `frontend/` is a Vite React app. `frontend/src/App.jsx` wires app state, settings, reader layout, search navigation, audio, and EPUB/PDF viewer selection; `frontend/src/api.js` centralizes API fetches; `frontend/src/main.jsx` is the entrypoint; `frontend/src/App.css` contains app-specific reader/reflow layout additions.
- Project structure: `frontend/src/components/` contains the main UI surfaces: `Welcome.jsx`, `Sidebar.jsx`, `PdfViewer.jsx`, `ReflowViewer.jsx`, `Pill.jsx`, and `icons.jsx`. `frontend/src/hooks/` contains `useBookState.js` for book/page loading and `useAudioPlayback.js` for playback/preload behavior.
- Project structure: shared styling is split across `frontend/src/styles/tokens.css` (theme variables), `welcome.css`, `reader.css`, `page-turn.css`, and `pill.css`, imported by `frontend/src/index.css`. Dark-mode palette changes should generally start in `tokens.css`, with only targeted component overrides elsewhere.
- App launch/build: root `launcher.pyw` starts the app; `.codex/environments/environment.toml` installs backend requirements, runs `npm install`/`npm run build` in `frontend/`, then launches `launcher.pyw`.
- EPUB reader paper geometry is intentionally fixed. Do not make page width, page height, margins, header/footer, or text area responsive; only switch between one fixed page and two fixed pages based on available width.
- Page-turn animation now uses `frontend/src/styles/page-turn.css` with `pt-*` classes rendered inside `ReflowViewer` in two-page mode only.
- Known follow-up: the animated page faces currently show text, but the wrong source/destination pages can appear during turns. The next animation pass should correct front/back face page selection so the flipping sheet text exactly matches the visible reader page being turned and the destination page behind it.
